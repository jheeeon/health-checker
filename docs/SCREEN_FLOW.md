# Screen Flow

## 문서 목적

이 문서는 `health-checker` MVP의 한 페이지 안에서 필요한 화면 영역, 상태, 사용자 흐름을 정리한다.

다음 작업의 Codex는 이 문서를 읽고 아래를 바로 이해할 수 있어야 한다.

- 한 장짜리 페이지에 어떤 영역이 필요한가
- 각 영역이 어떤 데이터를 보여주는가
- 사용자가 어떤 액션으로 상태를 변경하는가

## 설계 기준

- 전체 앱은 단일 페이지다.
- 별도 라우팅은 도입하지 않는다.
- GitHub Pages에서 동작하는 정적 웹앱으로 만든다.
- Supabase는 체크 상태와 기록 저장에만 사용한다.
- 모든 핵심 기능은 모바일 화면에서도 접근 가능해야 한다.

## 페이지 영역

### 1. 상단 요약 영역

목적:

- 오늘의 일차, 전체 진행률, 저장 상태를 빠르게 보여준다.

핵심 요소:

- 앱 제목
- 오늘 일차 표시
- 프로그램 시작일
- 전체 완료율
- 저장 상태 배지

상태 표현:

- `불러오는 중`
- `저장 중`
- `저장됨`
- `저장 실패`
- `오프라인 또는 연결 실패`

### 2. 오늘의 목표 영역

목적:

- 사용자가 페이지를 열었을 때 오늘 해야 할 일을 바로 확인하게 한다.

핵심 요소:

- 오늘 일차 제목
- 오늘의 핵심 목표
- 체크 항목 목록
- 간단한 주의 문구

핵심 액션:

- 체크 항목 선택/해제
- 변경 즉시 저장

### 3. 7일 프로그램 영역

목적:

- 전체 7일 계획을 한눈에 보고, 각 일차의 완료 상태를 확인한다.

핵심 요소:

- 1일차부터 7일차까지의 카드 또는 섹션
- 일차별 제목
- 일차별 운동 내용
- 일차별 체크 항목
- 완료 상태 표시

상태 표현 규칙:

- 오늘 일차는 시각적으로 강조한다.
- 완료한 일차는 차분한 완료 표시를 한다.
- 미래 일차도 열람과 체크를 막지 않는다.
- 과거 일차도 수정 가능하다.

### 4. 회복 기록 영역

목적:

- 체중이나 운동량보다 회복감을 기록하게 한다.

핵심 요소:

- 피로감 점수
- 집중력 점수
- 잠의 질 점수
- 몸의 안정감 점수
- 자유 메모

권장 UI:

- 점수는 1부터 5까지의 segmented control 또는 버튼 그룹
- 메모는 짧은 textarea
- 저장은 자동 저장을 기본으로 하되 저장 상태를 표시

### 5. 점검일 질문 영역

목적:

- 7일차에는 프로그램 반복 가능성을 중심으로 회고하게 한다.

핵심 요소:

- 아침 몸무게
- 피로감
- 집중력
- 잠의 질
- 계단 체감
- 몸의 안정감
- 식욕 변화
- 기분 변화
- `이번 주 프로그램을 다시 반복할 수 있을 것 같은가?`

표시 규칙:

- 7일차 카드 안에 포함한다.
- 다른 일차에서도 미리 열람할 수 있다.

## 주요 사용자 흐름

### 페이지 최초 진입

1. 사용자가 GitHub Pages URL을 연다.
2. 페이지가 Supabase 연결 정보를 사용해 데이터를 조회한다.
3. 저장된 `program_start_date`와 `daily_logs`를 불러온다.
4. 데이터가 없으면 기본 상태를 생성한다.
5. 오늘 일차와 체크 상태를 표시한다.

### 체크 저장

1. 사용자가 체크 항목을 누른다.
2. UI는 즉시 체크 상태를 반영한다.
3. 저장 상태를 `저장 중`으로 바꾼다.
4. Supabase 저장 성공 시 `저장됨`으로 바꾼다.
5. 실패 시 체크 상태는 유지하되 `저장 실패`를 표시한다.

### 다른 기기에서 열기

1. 사용자가 같은 URL을 다른 기기에서 연다.
2. 페이지가 Supabase에서 최신 상태를 불러온다.
3. 이전 기기에서 저장한 체크 상태와 메모가 표시된다.

### 저장 실패 후 복구

1. 저장 실패 상태가 표시된다.
2. 사용자는 새로고침하거나 다시 체크를 변경한다.
3. 다음 저장 요청이 성공하면 원격 상태가 갱신된다.

## 반응형 레이아웃 원칙

### 모바일

- 오늘의 목표를 가장 위에 배치한다.
- 7일 프로그램은 세로 카드 목록으로 표시한다.
- 체크 영역은 엄지로 누르기 쉬운 높이를 확보한다.
- 긴 설명문은 접기/펼치기 없이 읽을 수 있되 과도하게 좁아지지 않게 한다.

### 데스크톱

- 상단 요약은 가로로 배치한다.
- 오늘의 목표와 회복 기록을 상단에서 함께 볼 수 있게 한다.
- 7일 프로그램은 2열 또는 3열 카드 그리드를 고려한다.
- 카드 안 텍스트가 잘리지 않게 한다.

## 화면 밖 기능

MVP에서는 아래 화면을 만들지 않는다.

- 로그인 화면
- 설정 화면
- 관리자 화면
- 통계 상세 화면
- 별도 기록 히스토리 화면
