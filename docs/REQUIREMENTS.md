# Health Checker Requirements v1

## 문서 목적

이 문서는 `health-checker`의 첫 구현 범위를 고정하기 위한 요구사항 명세서다.

다음 작업의 Codex는 이 문서를 기준으로 아래를 빠르게 판단할 수 있어야 한다.

- 지금 당장 구현해야 하는 기능은 무엇인가
- 첫 버전에서 의도적으로 제외할 기능은 무엇인가
- 화면과 데이터 구조가 어떤 목표를 위해 존재하는가

## 제품 목표

사용자가 7일 운동 회복 프로그램을 매일 확인하고, 수행 여부와 몸 상태를 여러 기기에서 이어서 기록할 수 있게 한다.

핵심은 `운동 성과 관리`가 아니라 `회복 가능한 생활 리듬 확인`이다.

## 사용자 배경

- 사용자는 Android 개발 경험이 있다.
- 웹 개발 및 웹 배포 경험은 많지 않다.
- 복잡한 웹앱보다 개인적으로 매일 열어볼 수 있는 단순한 페이지를 원한다.
- 첫 버전은 보안보다 빠른 구현과 실사용 검증을 우선한다.

## 핵심 원칙

- 한 장짜리 페이지로 만든다.
- 모바일과 데스크톱에서 모두 보기 좋게 동작한다.
- 매일의 체크 상태는 브라우저가 아니라 원격 저장소에 저장한다.
- 같은 URL을 여러 기기에서 열면 같은 진행 상태가 보여야 한다.
- 운동 강도를 올리는 도구가 아니라 회복감과 지속 가능성을 확인하는 도구로 설계한다.

## MVP 사용자 시나리오

1. 사용자가 GitHub Pages에 배포된 페이지를 연다.
2. 페이지가 Supabase에서 현재 7일 프로그램 상태를 불러온다.
3. 사용자는 오늘의 운동 목표와 회복 체크 항목을 확인한다.
4. 사용자는 완료한 항목을 체크한다.
5. 사용자는 필요하면 간단한 메모나 몸 상태 점수를 남긴다.
6. 페이지는 변경 내용을 Supabase에 저장한다.
7. 사용자가 다른 기기에서 같은 URL을 열면 저장된 상태가 반영된다.

## MVP 포함 범위

첫 버전에 반드시 포함할 기능:

- 7일 운동 프로그램 표시
- 각 날짜별 목표와 체크 항목 표시
- 체크 상태 저장 및 불러오기
- Supabase 기반 기기 간 동기화
- 오늘 날짜 기준으로 현재 일차 강조
- 전체 진행률 표시
- 간단한 회복 기록 입력
  - 피로감
  - 집중력
  - 잠의 질
  - 몸의 안정감
  - 자유 메모
- 모바일/데스크톱 반응형 레이아웃
- 저장 중, 저장 완료, 저장 실패 상태 표시
- GitHub Pages 정적 배포 가능 구조

## MVP 제외 범위

첫 버전에서 의도적으로 제외할 기능:

- 로그인
- 사용자별 계정 분리
- 비공개 데이터 보호
- 운동 알림/푸시
- 캘린더 연동
- 그래프 기반 장기 통계
- AI 코칭
- 소셜 공유
- 관리자 화면
- 서버 직접 운영
- 결제 또는 유료 인프라

## 운동 프로그램 콘텐츠 범위

초기 콘텐츠는 사용자가 제공한 7일 프로그램을 기준으로 한다.

- 1일차: 회복 시작일
- 2일차: 아주 약한 근력
- 3일차: 회복 유산소
- 4일차: 휴식 같은 활동일
- 5일차: 약한 근력 + 회복
- 6일차: 긴 저강도 이동
- 7일차: 점검일

콘텐츠는 의료 처방이 아니라 일반 건강 정보로 다룬다.

## UX 원칙

- 첫 화면에서 오늘 할 일을 바로 알 수 있어야 한다.
- 체크는 한 번의 탭 또는 클릭으로 끝나야 한다.
- 완료 여부보다 몸 상태 기록이 눈에 들어와야 한다.
- 사용자가 과하게 운동하도록 자극하는 표현을 피한다.
- 화면 설명문은 최소화하고, 실제 기록 인터페이스를 중심에 둔다.
- 저장 실패 시 사용자가 현재 상태를 잃어버렸다고 느끼지 않게 한다.

## 완료 판정

아래가 가능하면 MVP 요구사항은 충족된 것으로 본다.

- GitHub Pages URL로 페이지를 열 수 있다.
- 모바일과 데스크톱에서 화면이 깨지지 않는다.
- 체크 상태가 Supabase에 저장된다.
- 다른 기기 또는 다른 브라우저에서 같은 체크 상태를 볼 수 있다.
- 저장 실패 상황에서 사용자에게 상태가 표시된다.
- 7일 프로그램 전체를 한 페이지 안에서 확인할 수 있다.
