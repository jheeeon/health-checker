# QA Test Scenarios

## 문서 목적

이 문서는 `health-checker` MVP를 수동 검증할 때 사용할 테스트 시나리오 시트다.

목표:

- 한 장짜리 웹페이지의 정상 동작을 빠짐없이 점검한다.
- Supabase 저장과 기기 간 동기화를 검증한다.
- 회귀 테스트 시 동일한 기준으로 반복 검증할 수 있게 한다.

## 테스트 범위

현재 MVP 구현 기준 범위:

- GitHub Pages 정적 페이지 로딩
- Supabase 데이터 불러오기
- 체크 상태 저장
- 회복 기록 저장
- 다른 브라우저 또는 다른 기기에서 동기화 확인
- 모바일/데스크톱 반응형 레이아웃
- 저장 실패 상태 표시

현재 제외:

- 로그인
- 사용자별 데이터 분리
- 비공개 접근 제어
- 알림
- 장기 통계
- 캘린더 연동

## 권장 테스트 환경

- 데스크톱 브라우저: Chrome 또는 Safari 최신 버전
- 모바일 브라우저: iPhone Safari 또는 Android Chrome
- 배포 환경: GitHub Pages
- 데이터 저장소: Supabase Free 프로젝트

## 기본 테스트 데이터

권장 준비:

- Supabase `app_state` 테이블
- Supabase `daily_logs` 테이블
- 1일부터 7일까지의 운동 프로그램 콘텐츠
- 테스트 전 초기 상태 또는 기존 상태 백업

## 결과 기록 규칙

각 시나리오는 아래 중 하나로 기록한다.

- `PASS`
- `FAIL`
- `BLOCKED`
- `N/A`

권장 기록 항목:

- 실행 일시
- 실행 브라우저/기기
- 배포 URL
- 결과
- 실제 결과 메모
- 스크린샷 경로 또는 첨부 여부

## 테스트 시나리오

### QA-001 페이지 첫 로딩

- 목적: GitHub Pages URL에서 페이지가 정상 표시되어야 한다.
- 사전조건: 배포 완료
- 절차:
  1. 배포 URL 접속
  2. 첫 화면 확인
- 기대결과:
  - 앱 제목이 보인다.
  - 오늘의 목표 영역이 보인다.
  - 7일 프로그램 영역이 보인다.
  - 콘솔에 치명적인 JavaScript 오류가 없다.

### QA-002 Supabase 초기 데이터 생성

- 목적: 저장된 데이터가 없을 때 기본 상태가 생성되어야 한다.
- 사전조건: Supabase 테이블이 비어 있음
- 절차:
  1. 페이지 접속
  2. 로딩 완료 대기
  3. Supabase 테이블 확인
- 기대결과:
  - `app_state`에 `default` 레코드가 생성된다.
  - `daily_logs`에 1일부터 7일까지 레코드가 생성된다.
  - 화면에 오늘 일차가 표시된다.

### QA-003 체크 항목 저장

- 목적: 체크 변경이 Supabase에 저장되어야 한다.
- 사전조건: 페이지 로딩 완료
- 절차:
  1. 오늘 목표의 체크 항목 하나를 선택
  2. 저장 상태 확인
  3. Supabase `daily_logs.checked_items` 확인
- 기대결과:
  - UI 체크 상태가 즉시 바뀐다.
  - 저장 상태가 `저장 중` 후 `저장됨`으로 바뀐다.
  - Supabase에 변경 값이 저장된다.

### QA-004 체크 항목 해제 저장

- 목적: 완료 체크를 해제해도 원격 상태가 갱신되어야 한다.
- 사전조건: 체크된 항목이 1개 이상 있음
- 절차:
  1. 체크된 항목을 다시 선택해 해제
  2. 저장 완료 대기
  3. 페이지 새로고침
- 기대결과:
  - 해제 상태가 유지된다.
  - 새로고침 후에도 해제 상태로 표시된다.

### QA-005 회복 점수 저장

- 목적: 피로감, 집중력, 잠의 질, 몸의 안정감 점수가 저장되어야 한다.
- 사전조건: 페이지 로딩 완료
- 절차:
  1. 각 점수를 선택
  2. 저장 완료 대기
  3. 페이지 새로고침
- 기대결과:
  - 선택한 점수가 유지된다.
  - Supabase 해당 필드가 갱신된다.

### QA-006 메모 저장

- 목적: 자유 메모가 저장되어야 한다.
- 사전조건: 페이지 로딩 완료
- 절차:
  1. 메모 입력
  2. 저장 완료 대기
  3. 페이지 새로고침
- 기대결과:
  - 메모 내용이 유지된다.
  - Supabase `memo` 필드가 갱신된다.

### QA-007 다른 브라우저 동기화

- 목적: 같은 기기 안의 다른 브라우저에서도 저장 상태가 보여야 한다.
- 사전조건: Chrome과 Safari 사용 가능
- 절차:
  1. Chrome에서 체크 항목 변경
  2. 저장 완료 대기
  3. Safari에서 같은 URL 접속
- 기대결과:
  - Safari에서 Chrome에서 저장한 체크 상태가 보인다.

### QA-008 다른 기기 동기화

- 목적: 모바일과 데스크톱 사이에 체크 상태가 동기화되어야 한다.
- 사전조건: 두 기기 모두 인터넷 연결 가능
- 절차:
  1. 데스크톱에서 체크 항목 변경
  2. 저장 완료 대기
  3. 모바일에서 같은 URL 접속 또는 새로고침
- 기대결과:
  - 모바일에서 데스크톱 변경 사항이 보인다.

### QA-009 저장 실패 표시

- 목적: Supabase 저장 실패 시 사용자에게 상태가 표시되어야 한다.
- 사전조건: 네트워크 차단 또는 잘못된 Supabase 설정
- 절차:
  1. 저장이 실패하도록 환경 구성
  2. 체크 항목 변경
- 기대결과:
  - UI가 멈추지 않는다.
  - 저장 실패 상태가 표시된다.
  - 사용자가 현재 체크 상태를 눈으로 확인할 수 있다.

### QA-010 모바일 레이아웃

- 목적: 모바일 화면에서 핵심 기능이 잘 보여야 한다.
- 사전조건: 모바일 브라우저 또는 모바일 viewport
- 절차:
  1. 페이지 접속
  2. 상단부터 하단까지 스크롤
  3. 체크 항목과 점수 입력 조작
- 기대결과:
  - 텍스트가 잘리지 않는다.
  - 버튼/체크 영역이 누르기 어렵지 않다.
  - 가로 스크롤이 생기지 않는다.

### QA-011 데스크톱 레이아웃

- 목적: 데스크톱 화면에서 정보 밀도가 적절해야 한다.
- 사전조건: 데스크톱 브라우저
- 절차:
  1. 페이지 접속
  2. viewport 너비를 크게 조정
- 기대결과:
  - 화면이 지나치게 넓게 퍼지지 않는다.
  - 오늘 목표와 기록 영역이 찾기 쉽다.
  - 7일 카드들이 균형 있게 배치된다.

### QA-012 오늘 일차 강조

- 목적: 프로그램 시작일 기준 오늘 일차가 강조되어야 한다.
- 사전조건: `app_state.program_start_date` 설정 가능
- 절차:
  1. 시작일을 오늘 기준 2일 전으로 설정
  2. 페이지 새로고침
- 기대결과:
  - 3일차가 오늘 일차로 강조된다.
  - 상단 요약에도 3일차가 표시된다.
